Estructura geométrica a definir en el TXT:
Quiero una estructura sencilla en el centro del mundo.

Dimensiones:

* 10 x 10 x 10 voxels
* mismo tamaño de voxel que hemos venido usando: 0.1m por voxel

Forma:

* 4 columnas, una en cada esquina
* cada columna es de 1 voxel x 1 voxel en planta
* cada columna sube hasta completar la altura total de 10 voxels
* arriba del todo hay un “techo/toldo”
* ese techo ocupa la planta 10x10
* pero el techo debe ser en patrón tablero:

  * voxel sí, voxel no, voxel sí, voxel no
  * alternado
* el interior debe quedar vacío
* quiero ver esa estructura centrada en el medio del escenario

En resumen:

* una especie de estructura abierta con 4 columnas en las esquinas
* y un techo superior ajedrezado de voxels alternos

TXT source of truth:
Quiero que generes un fichero TXT con un formato sencillo, editable y claro.
No hace falta que copies el formato de Godot al 100%, pero sí quiero la misma filosofía:

* texto
* voxelado
* fácil de entender
* capas si tiene sentido

Puedes elegir tú el formato más práctico para esta V2, pero debe cumplir:

* ser humano-legible
* representar la estructura anterior
* servir como source of truth editable
* ser parseable fácilmente desde Python

Python preprocess:
Quiero un script Python real que:

1. lea el TXT fuente
2. interprete la estructura voxel
3. genere un runtime artifact pensado para Sokol, no para Godot
4. deje el runtime artifact escrito dentro del proyecto

Muy importante:
El runtime artifact debe estar orientado a Sokol.

No quiero una estructura pensada como `ArrayMesh`.
Quiero algo más cercano a lo que Sokol necesita.

Idealmente el runtime artifact debería contener algo así:

* metadatos del mundo
* vertex format
* lista de vértices
* lista de índices
* si hace falta, submeshes o bloques por material/color

No hace falta hacer un sistema complejo de materiales.
Con un color simple uniforme basta.
Pero el formato debe ser sensato para subirlo luego a buffers de Sokol.

Geometría:

* no renderizar un cubo por voxel como draw call separada
* el preprocess debe generar geometría útil para runtime
* puede ser una malla triangulada simple
* si quieres, puedes emitir quads y en runtime convertirlos a vértices/índices
* o directamente emitir vértices/índices finales desde Python
* elige el approach más natural para Sokol y más rápido de implementar

Mi preferencia:
Si ves claro generar ya en Python:

* vertices
* indices
  entonces hazlo así, porque Sokol trabaja más cerca de buffers.

Runtime en Sokol:
La app V2 debe:

1. cargar el runtime artifact generado por Python
2. parsearlo
3. crear vertex buffer e index buffer
4. dibujar la geometría
5. permitir mover la cámara en first-person
